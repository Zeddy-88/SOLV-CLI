# Product Spec — 대표 제출용 PDF 보고서

**Route:** `/dashboard/[id]/report`  
**대상 사용자:** 로그인한 영업사원  
**진입 경로:** 브리핑 화면(`/dashboard/[id]`)의 "대표 제출용 보고서 만들기" 버튼

---

## 목적

영업사원이 대표와의 미팅 자리에서 직접 건낼 수 있는 인쇄용 자료.  
복잡한 재무 데이터를 걷어내고, **"우리 회사에 이런 문제가 있고, 이렇게 해결할 수 있다"** 는 메시지에 집중.

영업 멘트는 포함하지 않는다. 이 문서는 대표가 읽는 자료이지, 영업사원의 참고 자료가 아니다.

---

## 미리보기 화면 (`/dashboard/[id]/report`)

PDF를 다운로드하기 전에 웹 브라우저에서 내용을 확인하는 화면.

**구성:**
- 상단 액션 바: "PDF 다운로드" 버튼, "← 브리핑으로 돌아가기" 링크
- 하단: A4 비율의 PDF 미리보기 렌더링 (`@react-pdf/renderer` 결과물)

---

## PDF 문서 스펙

### 기본 설정

| 항목 | 값 |
|---|---|
| 용지 크기 | A4 (210 × 297mm) |
| 분량 | 1~2페이지 (콘텐츠 양에 따라 자동) |
| 여백 | 상하 20mm, 좌우 18mm |
| 기본 폰트 | Noto Sans KR (한글 지원) |
| 생성 방식 | `@react-pdf/renderer` 서버사이드 렌더링 |

---

### 페이지 구성

#### 헤더 (모든 페이지 공통)

```
┌────────────────────────────────────────┐
│  (주)강서기전 재무 분석 보고서            │
│  2024년 12월 결산 기준 · 2026.03.25 작성 │
└────────────────────────────────────────┘
```

- 좌측: 기업명 + "재무 분석 보고서"
- 우측: 결산 기준일 + 보고서 작성일
- 하단 구분선

---

#### 섹션 1 — 기업 개요

한 줄 소개 + 주요 기업 정보 테이블.

**포함 항목:**
- 기업명, 대표자, 업종
- 설립연도, 종업원수, 소재지
- 신용등급 (있는 경우)

---

#### 섹션 2 — 핵심 재무 현황

3개년 주요 수치를 간결한 테이블로 표시.

| 항목 | 2022 | 2023 | 2024 |
|---|---|---|---|
| 매출액 | | | |
| 영업이익 | | | |
| 영업이익률 | | | |
| 당기순이익 | | | |
| 부채비율 | | | |

단위 명시 (백만원, %).  
최신 연도 컬럼 배경색 연하게 강조.

---

#### 섹션 3 — 주요 발견 사항

AI가 도출한 핵심 이슈. **최대 3개**만 포함 (PDF 분량 관리).  
브리핑 화면에서 'critical' 이슈를 우선 선택하고, 부족하면 'warning'으로 채움.

각 항목:
- 번호 + 제목 (굵게)
- 설명 (수치 근거 포함, 2~3줄)

---

#### 섹션 4 — 컨설팅 제안 포인트

AI가 도출한 강점 기반 제안. **최대 2개**.  
"문제만 나열하고 끝나는" 보고서가 되지 않도록, 긍정적 가능성을 함께 제시.

형식:
- 제목 (굵게)
- 설명 (2~3줄)

---

#### 푸터 (마지막 페이지)

```
본 보고서는 제출된 재무 자료를 기반으로 AI가 분석한 참고 자료입니다.
```

페이지 번호 (우측 하단): `1 / 2`

---

## 다운로드

- 버튼 텍스트: "PDF 다운로드"
- 파일명 자동 생성: `{기업명}_재무분석보고서_{YYYYMMDD}.pdf`  
  예: `강서기전_재무분석보고서_20260325.pdf`
- 응답: `/api/report`에서 `application/pdf` 스트림 반환
- 다운로드 완료 시 브라우저 기본 저장 다이얼로그 사용

---

## 제외 항목 (의도적으로 넣지 않는 것)

| 항목 | 제외 이유 |
|---|---|
| 영업 추천 멘트 | 영업사원 전용 정보. 대표가 보는 자료에 부적절 |
| 전체 재무비율 목록 | 분량 과다. 핵심만 선별 |
| 업계 순위 상세 | 민감할 수 있음. 필요 시 영업사원이 구두로 전달 |
| AI 분석임을 강조하는 표현 | 신뢰도 저하 우려. 푸터에 최소한으로만 표기 |

---

## 오류 처리

| 상황 | 처리 |
|---|---|
| `analysis_result` 없음 (분석 미완료) | 미리보기 화면에서 "분석이 완료되지 않았습니다" 안내 후 브리핑 화면으로 이동 |
| PDF 생성 실패 | "보고서 생성 중 문제가 발생했어요. 다시 시도해주세요." + 재시도 버튼 |
| 타인의 분석 결과 접근 | 403 처리 후 `/history`로 리디렉트 |
