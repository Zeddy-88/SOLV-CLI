# Product Spec — PDF 업로드 화면

**Route:** `/`  
**대상 사용자:** 로그인한 영업사원  
**진입 경로:** 앱 첫 화면 / 네비게이션 "새 분석"

---

## 목적

영업사원이 기업 종합 보고서 PDF를 업로드하고 분석을 시작하는 화면.  
단순하고 빠르게. 업로드 → 분석 시작까지 3번의 클릭을 넘지 않는다.

---

## 화면 구성

```
┌─────────────────────────────────────┐
│  네비게이션 바                         │
├─────────────────────────────────────┤
│                                     │
│       업로드 드롭존                   │
│  (드래그앤드롭 또는 파일 선택)          │
│                                     │
├─────────────────────────────────────┤
│  지원 포맷 안내                        │
├─────────────────────────────────────┤
│  최근 분석 이력 (최대 3건)              │
└─────────────────────────────────────┘
```

---

## 섹션 상세 스펙

### 네비게이션 바

- 좌측: 앱 로고 / 서비스명
- 우측: 사용자 이메일 + 로그아웃 버튼, "분석 이력" 링크 (`/history`)

---

### 업로드 드롭존

화면 중앙에 크게 배치. 두 가지 업로드 방식 지원.

**기본 상태 (파일 미선택):**
- 점선 테두리 박스
- 아이콘: 파일 업로드 아이콘
- 메인 텍스트: "기업 종합 보고서 PDF를 업로드하세요"
- 서브 텍스트: "드래그앤드롭 하거나 클릭하여 파일을 선택하세요"
- 파일 조건 안내: "PDF · 최대 20MB"

**파일 드래그 오버 상태:**
- 테두리 색상 변경 (파란색)
- 배경색 연하게 변경
- 텍스트: "여기에 놓으세요"

**파일 선택 완료 상태 (업로드 전):**
- 파일명 표시
- 파일 크기 표시
- "분석 시작" 버튼 활성화 (파란색 주요 버튼)
- "다른 파일 선택" 링크 (버튼 아래 작게)

**업로드 & 분석 진행 중 상태:**
- 진행 단계를 순서대로 표시:
  1. "파일 업로드 중..." (스피너)
  2. "재무 데이터를 읽고 있어요..." (스피너)
  3. "AI가 분석하고 있어요. 잠깐만 기다려주세요." (스피너)
- 단계 전환은 Supabase Realtime `status` 변경에 따라 자동 업데이트
- 진행 중 다른 파일 선택 / 취소 불가 (버튼 비활성화)
- 분석 완료 시 `/dashboard/[id]`로 자동 이동

---

### 지원 포맷 안내

드롭존 하단에 간결하게 표시. 접었다 펼 수 있는 토글이면 더 좋음.

**표시 내용:**
- CRETOP / K0DATA 기업종합보고서 ✓
- NICE BizLine 기업분석보고서 ✓
- 일반 재무제표 PDF (손익계산서 · 재무상태표 포함) ✓

**주의 안내 (작은 글씨):**
- "재무 데이터가 없는 단순 소개 자료는 분석되지 않을 수 있습니다."

---

### 최근 분석 이력 (최대 3건)

과거 분석이 1건 이상 있을 때만 표시. 없으면 이 섹션 숨김.

섹션 제목: "최근 분석"

각 항목:
- 기업명
- 분석 기준일 (예: 2024-12-31 결산)
- 분석 완료 시각 (예: 3일 전)
- 클릭 시 해당 `/dashboard/[id]`로 이동

하단에 "전체 이력 보기 →" 링크 (`/history`)

## 유효성 검사

### 클라이언트 (즉시 피드백)

| 조건 | 오류 메시지 |
|---|---|
| PDF 파일이 아닌 경우 | "PDF 파일만 업로드할 수 있어요." |
| 파일 크기 20MB 초과 | "파일 크기가 너무 커요. 20MB 이하의 파일을 사용해주세요." |

### 서버 (API 응답 후 피드백)

| 조건 | 오류 메시지 |
|---|---|
| MIME 타입 불일치 | "올바른 PDF 파일이 아닙니다." |
| 파일 크기 초과 | "파일 크기 제한을 초과했습니다." |
| 텍스트 추출 실패 | "PDF에서 텍스트를 읽을 수 없어요. 스캔본이 아닌 텍스트 PDF를 사용해주세요." |
| 재무 데이터 없음 | "재무 데이터를 찾을 수 없어요. 재무제표가 포함된 PDF인지 확인해주세요." |
| 서버 오류 | "분석 중 문제가 발생했어요. 잠시 후 다시 시도해주세요." |

오류 발생 시 드롭존을 초기 상태로 되돌리고 오류 메시지를 드롭존 하단에 표시.  
"다시 시도" 링크 함께 제공.

---

## 스캔본 PDF 처리

`pdf-parse`로 텍스트를 추출했을 때 텍스트가 거의 없거나(100자 미만) 전부 이미지인 경우,
스캔본으로 판단하고 오류 처리.

OCR 지원은 Post-MVP 범위. MVP에서는 텍스트 기반 PDF만 지원.

---

## 비인증 접근

미로그인 상태에서 `/`에 접근하면 `/auth/login`으로 리디렉트.  
리디렉트 전 원래 경로를 쿼리 파라미터로 보존 (`?redirect=/`).  
로그인 완료 후 원래 경로로 복귀.
