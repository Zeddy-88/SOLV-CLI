# Product Spec — 인증 (로그인 / 회원가입)

**Routes:** `/auth/login`, `/auth/signup`  
**인증 방식:** 이메일 + 패스워드 (Supabase Auth)

---

## 로그인 화면 (`/auth/login`)

**구성:**
- 앱 로고 + 서비스명
- 이메일 입력
- 패스워드 입력
- "로그인" 버튼
- "아직 계정이 없으신가요? 회원가입" 링크 → `/auth/signup`

**동작:**
- 로그인 성공 → `?redirect=` 파라미터가 있으면 해당 경로로, 없으면 `/`로 이동
- 로그인 실패 → "이메일 또는 패스워드가 올바르지 않습니다." 인라인 오류 표시
- 입력값 유효성: 이메일 형식 검사, 패스워드 1자 이상

---

## 회원가입 화면 (`/auth/signup`)

**구성:**
- 앱 로고 + 서비스명
- 이름 입력 (선택 — 나중에 사이드바 하단 표시용)
- 이메일 입력
- 패스워드 입력 (최소 8자)
- 패스워드 확인 입력
- "회원가입" 버튼
- "이미 계정이 있으신가요? 로그인" 링크 → `/auth/login`

**동작:**
- 회원가입 성공 → `/auth/login` 리디렉트 (또는 자동 로그인 후 온보딩 웰컴 화면)
- 이메일 중복 → "이미 사용 중인 이메일입니다." 오류 표시
- 패스워드 불일치 → "패스워드가 일치하지 않습니다." 오류 표시

---

## 비인증 접근 처리

`middleware.ts`에서 처리. 보호된 경로(`/`, `/dashboard/*`, `/history`) 접근 시:

1. `/auth/login?redirect={원래경로}`로 리디렉트
2. 로그인 완료 후 `redirect` 파라미터 경로로 복귀

---

## 로그아웃

사이드바 하단 설정 메뉴 → "로그아웃" 클릭 시:
- Supabase `signOut()` 호출
- 세션 쿠키 삭제
- `/auth/login`으로 리디렉트

---

## 보안 유의사항

- 패스워드는 Supabase Auth가 자동으로 해싱. 서버 코드에서 직접 처리하지 않는다.
- 로그인 실패 메시지는 "이메일이 없습니다" / "패스워드가 틀렸습니다"로 구분하지 않는다. 계정 존재 여부 노출 방지.
- 세션은 `@supabase/ssr`의 쿠키 기반 방식으로 관리. 서버 컴포넌트에서 안전하게 읽힌다.
