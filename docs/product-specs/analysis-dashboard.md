# Product Spec — 분석 결과 브리핑 화면

**Route:** `/dashboard/[id]`  
**대상 사용자:** 로그인한 영업사원  
**진입 경로:** 업로드 완료 후 자동 이동 / 이력 목록에서 클릭

---

## 목적

영업사원이 대표와의 미팅 전, 혹은 미팅 자리에서 빠르게 참고할 수 있는 브리핑 화면.  
재무 데이터를 해석하고 "어떻게 말을 꺼낼지"까지 알려주는 것이 핵심.

---

## 화면 구성 (섹션 순서)

```
┌─────────────────────────────────────┐
│  1. 회사 헤더                         │
├─────────────────────────────────────┤
│  2. 핵심 지표 카드 (4개)               │
├───────────────────┬─────────────────┤
│  3. 재무진단 등급  │  4. 3개년 추이    │
├───────────────────┴─────────────────┤
│  5. 핵심 이슈                         │
├─────────────────────────────────────┤
│  6. 4인 전문가 분석                    │
├─────────────────────────────────────┤
│  7. 영업 추천 멘트                     │
├─────────────────────────────────────┤
│  8. PDF 보고서 생성 버튼               │
└─────────────────────────────────────┘
```

## 섹션 상세 스펙

### 1. 회사 헤더

| 항목 | 내용 |
|---|---|
| 회사명 | 크게 표시 (예: (주)강서기전) |
| 업종 | 표준산업분류명 (예: 기타 일반 기계 및 장비 수리업) |
| 종업원수 | N명 |
| 신용등급 | 배지 형태 (예: BBB-, bb) — 데이터가 있을 때만 표시 |
| 업계 순위 | 배지 형태 (예: 업계 220위) — 데이터가 있을 때만 표시 |
| WATCH 상태 | 배지 형태 (예: 정상) — NICE 포맷일 때만 표시 |
| 분석 기준일 | 결산 기준일 (예: 2024-12-31 결산 기준) |
| 보고서 출처 | CRETOP / NICE BizLine / 일반 재무제표 중 해당 항목 |

---

### 2. 핵심 지표 카드

4개 카드를 가로로 나열. 각 카드 구성:
- 지표명 (작은 라벨)
- 수치 (크게)
- 전년 대비 증감 (화살표 + %, 증가 파랑 / 감소 빨강)

| 카드 | 지표 | 표시 형식 |
|---|---|---|
| 연 매출 | 최신 연도 매출액 | N억 원 |
| 영업이익률 | 최신 연도 영업이익률 | N.N% |
| 당기순이익 | 최신 연도 당기순이익 | N백만 원 |
| 부채비율 | 최신 연도 부채비율 | N.N% |

**전년 대비 증감 계산:**
- 영업이익률 / 부채비율: 전년 대비 %p 변화
- 매출 / 순이익: 전년 대비 % 변화

---

### 3. 재무진단 등급

CRETOP 포맷 또는 NICE 포맷에서 재무진단 데이터가 존재할 때만 표시.  
데이터가 없으면 이 섹션은 렌더링하지 않음.

5개 항목을 가로로 나열:
- 성장성
- 수익성
- 재무구조
- 부채상환능력
- 활동성

각 항목: 항목명 + 등급 텍스트

**등급별 색상:**
| 등급 | 색상 |
|---|---|
| 우수 / 양호 | 초록 |
| 보통 | 노랑 |
| 보통이하 / 열위 | 빨강 |

---

### 4. 3개년 재무 추이

3개년(최신 3개 결산연도) 주요 수치를 테이블로 표시.

| 항목 | 연도1 | 연도2 | 연도3 |
|---|---|---|---|
| 매출 (억원) | | | |
| 영업이익 (백만원) | | | |
| 당기순이익 (백만원) | | | |
| 영업이익률 (%) | | | |
| ROE (%) | | | |
| 부채비율 (%) | | | |

최신 연도 컬럼은 볼드 강조.  
전년 대비 악화된 수치는 빨강으로 표시.

---

### 5. 핵심 이슈

AI가 도출한 이슈 목록. 최대 5개.  
심각도에 따라 두 가지 스타일로 구분:

**위험 (critical):** 왼쪽 빨간 세로선 + 연한 빨간 배경
- 제목 (굵게)
- 설명 (수치 근거 포함)

**주의 (warning):** 왼쪽 노란 세로선 + 연한 노란 배경
- 제목 (굵게)
- 설명 (수치 근거 포함)

이슈가 0개일 경우: "특별한 위험 이슈가 발견되지 않았습니다." 안내 메시지 표시.

---

### 6. 4인 전문가 분석

재무 데이터를 4가지 전문가 관점에서 분석한 결과.  
상단에 4인의 개별 의견 카드, 하단에 통합 의견을 배치한다.

#### 개별 의견 카드 (4개, 2×2 그리드)

각 카드 구성:
- 페르소나 아이콘 + 이름 (예: 재무 컨설턴트)
- 역할 한 줄 설명 (예: "숫자 뒤의 진짜 의미를 읽는다")
- 의견 본문 (3~5문장)
- 핵심 메시지 (굵게, 카드 하단)

**페르소나별 색상 구분:**

| 페르소나 | 색상 테마 |
|---|---|
| 재무 컨설턴트 | 파랑 계열 |
| 영업왕 | 초록 계열 |
| 커뮤니케이션 전문가 | 보라 계열 |
| 심리 전문가 | 주황 계열 |

#### 통합 의견

4개 카드 아래, 구분선으로 분리하여 배치.  
섹션 제목: "4인 전문가 통합 의견"  
본문: 2~3문단. 4인의 관점이 종합된 최종 인사이트.  
배경색: 연한 회색으로 다른 섹션과 시각적으로 구분.

---

### 7. 영업 추천 멘트

AI가 생성한 4단계 영업 스크립트. 4인 전문가의 통합 관점이 반영된 멘트.  
각 단계는 카드 형태로 표시.

| 단계 | 태그 | 목적 |
|---|---|---|
| 1단계 | 오프닝 | 대표의 공감을 유도하며 대화 시작 |
| 2단계 | 문제 구체화 | 재무 데이터 근거로 문제를 명확히 제시 |
| 3단계 | 위기감 조성 | 지금 조치하지 않으면 어떤 리스크가 있는지 |
| 4단계 | 클로징 | 컨설팅 계약의 필요성으로 자연스럽게 연결 |

각 카드 우측 상단에 **복사 버튼** 제공 (클릭 시 해당 멘트 클립보드에 복사).  
복사 완료 시 "복사됨" 토스트 메시지 표시 (2초 후 사라짐).

---

### 8. PDF 보고서 생성 버튼

화면 하단 고정 또는 섹션 7 아래에 배치.

- 버튼 텍스트: "대표 제출용 보고서 만들기"
- 클릭 시: `/dashboard/[id]/report` 페이지로 이동
- 부가 설명: "영업 멘트를 제외한 재무 요약본 · A4 1~2페이지"

---

## 로딩 상태

분석이 완료되기 전(`status: 'pending' | 'processing'`) 이 화면에 접근하면
분석 진행 상태를 단계별로 표시한다.

| 상태 | 표시 메시지 |
|---|---|
| pending | "파일을 업로드하고 있어요..." |
| processing (파싱 중) | "재무 데이터를 읽고 있어요..." |
| processing (AI 분석 중) | "AI가 분석하고 있어요. 잠깐만 기다려주세요." |
| done | 브리핑 화면 렌더링 |
| error | 오류 메시지 + 재시도 버튼 |

상태는 Supabase Realtime 구독으로 실시간 업데이트.

---

## 오류 처리

| 상황 | 처리 |
|---|---|
| 분석 실패 (`status: 'error'`) | 오류 메시지 표시 + "다시 분석하기" 버튼 |
| 존재하지 않는 `id` | 404 페이지 |
| 타인의 분석 결과 접근 | 403 처리 후 `/history`로 리디렉트 |
| 재무진단 데이터 없음 | 섹션 3 숨김 (오류 아님) |
| 업계 순위 데이터 없음 | 해당 배지 숨김 (오류 아님) |

---

## 반응형

- 기본 타겟: 데스크톱 (1280px 이상) — 영업사원이 노트북에서 주로 사용
- 모바일 대응은 Post-MVP 범위. MVP에서는 모바일 레이아웃 최적화 불필요.
- 단, 텍스트와 버튼이 모바일에서 깨지지 않는 수준의 기본 대응은 유지.
