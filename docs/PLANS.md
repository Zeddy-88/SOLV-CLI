# PLANS.md — 개발 계획 및 마일스톤

## 개발 방식

하네스 엔지니어링 (AI 에이전트 기반 개발).  
각 Phase는 독립적으로 동작 가능한 상태로 완료되어야 한다.  
Phase가 끝날 때마다 실제로 배포하고 직접 사용해보며 다음 Phase를 조정한다.

---

## 전체 로드맵

```
Phase 1 — 기반 세팅       인증 + 프로젝트 골격
Phase 2 — 파싱 엔진       PDF 텍스트 추출 + 포맷별 파서
Phase 3 — AI 분석         Claude API 연동 + 분석 결과 저장
Phase 4 — 브리핑 화면     분석 결과 시각화 UI
Phase 5 — PDF 보고서      대표 제출용 PDF 생성
Phase 6 — 마무리          이력 관리 + 안정화 + 배포
```

---

## Phase 1 — 기반 세팅

**목표:** 로그인한 사용자가 앱에 접근할 수 있는 상태

### 작업 목록

- [ ] Next.js 14 프로젝트 생성 (App Router, TypeScript, Tailwind)
- [ ] Supabase 프로젝트 생성 및 환경변수 설정
- [ ] `@supabase/ssr` 연동 (서버/클라이언트 클라이언트 분리)
- [ ] 미들웨어 설정 — 비인증 접근 시 `/auth/login` 리디렉트
- [ ] `/auth/login` 페이지 — 이메일/패스워드 로그인
- [ ] `/auth/signup` 페이지 — 회원가입
- [ ] 로그인 후 `/`로 리디렉트
- [ ] 네비게이션 바 컴포넌트 (로고, 이력 링크, 로그아웃)
- [ ] Vercel 연동 및 첫 배포

**완료 기준:** 회원가입 → 로그인 → 메인 화면 진입 → 로그아웃이 정상 동작

---

## Phase 2 — 파싱 엔진

**목표:** PDF를 업로드하면 재무 데이터를 정규화된 형태로 추출할 수 있는 상태

### 작업 목록

- [ ] `analyses` 테이블 생성 및 RLS 설정
- [ ] Supabase Storage 버킷 생성 (`reports`) 및 접근 정책 설정
- [ ] `/api/upload` — 파일 업로드 + Storage 저장 + analyses 레코드 생성
- [ ] `pdf-parse` 설치 및 텍스트 추출 유틸 작성 (`lib/parsers/extract.ts`)
- [ ] 포맷 감지 로직 (`lib/parsers/detect.ts`)
- [ ] CRETOP 파서 (`lib/parsers/cretop.ts`)
- [ ] NICE BizLine 파서 (`lib/parsers/nice.ts`)
- [ ] 공통 정규화 타입 정의 (`types/financial.ts`)
- [ ] 파서 유닛 테스트 (샘플 PDF 2종 기준)
- [ ] 업로드 화면 UI (`/`) — 드롭존, 상태 표시, 유효성 검사

**완료 기준:** CRETOP / NICE BizLine PDF를 각각 업로드했을 때 `FinancialData` 타입으로 정상 추출됨. 스캔본 PDF는 오류 처리됨.

---

## Phase 3 — AI 분석

**목표:** 파싱된 재무 데이터를 Claude API로 분석하고 결과를 DB에 저장하는 상태

### 작업 목록

- [ ] Anthropic SDK 설치 및 환경변수 설정
- [ ] Claude 분석 프롬프트 작성 (`lib/claude/prompts.ts`)
- [ ] `AnalysisResult` 타입 정의 (`types/analysis.ts`)
- [ ] `/api/analyze` — Claude API 호출 + JSON 파싱 + DB 저장
- [ ] `analyses.status` 실시간 업데이트 (pending → processing → done/error)
- [ ] Supabase Realtime 구독 훅 (`hooks/useAnalysisStatus.ts`)
- [ ] 업로드 화면에 실시간 진행 상태 연동
- [ ] 분석 완료 시 `/dashboard/[id]`로 자동 이동
- [ ] Claude 응답 파싱 실패 시 재시도 로직 (최대 2회)
- [ ] Vercel `maxDuration = 60` 설정 (`/api/analyze`)

**완료 기준:** CRETOP / NICE PDF 업로드 후 60초 이내에 `AnalysisResult` JSON이 DB에 저장됨. 회사 요약, 이슈 목록, 영업 멘트 4단계가 모두 포함됨.

---

## Phase 4 — 브리핑 화면

**목표:** 영업사원이 분석 결과를 한눈에 파악하고 영업 멘트를 확인할 수 있는 상태

### 작업 목록

- [ ] `/dashboard/[id]` 페이지 라우트 및 데이터 fetching
- [ ] 회사 헤더 컴포넌트 (기업명, 신용등급 배지, 업계순위 배지)
- [ ] 핵심 지표 카드 4개 (매출, 영업이익률, 순이익, 부채비율 + 전년 대비)
- [ ] 3개년 재무 추이 테이블
- [ ] 재무진단 등급 컴포넌트 (데이터 없을 시 숨김 처리)
- [ ] 핵심 이슈 리스트 (위험/주의 스타일 분리)
- [ ] 영업 추천 멘트 카드 4단계 + 복사 버튼
- [ ] 복사 완료 토스트 메시지
- [ ] 분석 진행 중 로딩 상태 화면 (Realtime 연동)
- [ ] 오류 상태 화면 + 재시도 버튼
- [ ] "대표 제출용 보고서 만들기" 버튼

**완료 기준:** 브리핑 화면에서 모든 섹션이 정상 렌더링됨. 멘트 복사 기능 동작. 데이터 없는 섹션은 숨겨짐.

---

## Phase 5 — PDF 보고서

**목표:** 영업사원이 대표에게 건낼 수 있는 PDF를 다운로드할 수 있는 상태

### 작업 목록

- [ ] `@react-pdf/renderer` 설치 및 한글 폰트(Noto Sans KR) 설정
- [ ] PDF 문서 컴포넌트 작성 (`components/report/`)
  - [ ] 헤더 (기업명, 결산 기준일, 작성일)
  - [ ] 기업 개요 섹션
  - [ ] 핵심 재무 현황 테이블
  - [ ] 주요 발견 사항 (최대 3개)
  - [ ] 컨설팅 제안 포인트 (최대 2개)
  - [ ] 푸터 + 페이지 번호
- [ ] `/api/report` — PDF 스트림 생성 및 반환
- [ ] `/dashboard/[id]/report` — 미리보기 화면 + 다운로드 버튼
- [ ] 파일명 자동 생성 (`{기업명}_재무분석보고서_{YYYYMMDD}.pdf`)

**완료 기준:** 다운로드된 PDF가 A4 1~2페이지로 정상 출력됨. 한글 깨짐 없음. 영업 멘트가 포함되지 않음.

## Phase 6 — 마무리

**목표:** 실제 영업사원이 일상적으로 사용할 수 있는 수준의 완성도

### 작업 목록

- [ ] `/history` — 분석 이력 목록 페이지
- [ ] 메인 화면 최근 이력 3건 표시
- [ ] 에러 바운더리 및 전역 오류 처리
- [ ] 로딩 스켈레톤 UI (브리핑 화면 초기 로딩)
- [ ] 숫자 포맷팅 통일 (억/백만원 단위, 천 단위 콤마)
- [ ] 메타데이터 및 파비콘 설정
- [ ] 환경변수 Vercel 프로덕션 등록
- [ ] 최종 E2E 테스트 (CRETOP / NICE 각 1건 전체 플로우)
- [ ] 프로덕션 배포 확인

**완료 기준:** CRETOP / NICE PDF를 각각 업로드하여 브리핑 화면 확인, PDF 다운로드, 이력 조회까지 전체 플로우가 오류 없이 동작함.

## Post-MVP 백로그

우선순위 순으로 정리. MVP 완료 후 순차적으로 진행.

| 항목 | 설명 |
|---|---|
| 일반 재무제표 포맷 지원 | generic 파서 구현 — CRETOP/NICE 외 PDF 지원 |
| 영업 멘트 커스터마이징 | 영업사원이 생성된 멘트를 직접 수정하고 저장 |
| 모바일 최적화 | 태블릿/스마트폰에서 브리핑 화면 사용 가능하도록 |
| 분석 결과 공유 링크 | 팀 내 분석 결과 공유용 링크 생성 |
| 팀 계정 | 같은 조직의 영업사원들이 이력을 공유하는 팀 워크스페이스 |
| OCR 지원 | 스캔본 PDF에서도 텍스트 추출 가능하도록 |
| 분석 재실행 | 기존 업로드 파일을 다시 분석 (프롬프트 개선 후 재생성) |
