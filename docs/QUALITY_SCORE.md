# QUALITY_SCORE.md — 품질 기준

에이전트가 기능을 구현하거나 PR을 완료했을 때 스스로 점검하는 기준.  
아래 항목을 모두 통과해야 "완료"로 간주한다.

---

## 기능 품질

### 핵심 플로우 동작 확인

- [ ] PDF 업로드 → 파싱 → AI 분석 → 브리핑 화면까지 오류 없이 동작한다
- [ ] CRETOP 포맷 PDF로 전체 플로우가 동작한다
- [ ] NICE BizLine 포맷 PDF로 전체 플로우가 동작한다
- [ ] 분석 완료 후 `/dashboard/[id]`로 자동 이동된다
- [ ] PDF 보고서 다운로드가 정상 동작한다

### 예외 처리 확인

- [ ] 스캔본 PDF 업로드 시 명확한 오류 메시지가 표시된다
- [ ] 20MB 초과 파일 업로드 시 클라이언트에서 즉시 차단된다
- [ ] 분석 실패(`status: error`) 시 재시도 버튼이 표시된다
- [ ] 존재하지 않는 `analysisId` 접근 시 404 처리된다
- [ ] 타인의 분석 결과 접근 시 403 처리된다
- [ ] 네트워크 오류 시 빈 화면이 아닌 오류 메시지가 표시된다

### 로딩 상태

- [ ] 분석 진행 중 로딩 화면이 표시된다 (침묵하지 않는다)
- [ ] 브리핑 화면 초기 로딩 시 스켈레톤 UI가 표시된다
- [ ] 60초 이상 응답 없을 때 타임아웃 처리가 된다

## UI / UX 품질

### 데이터 표시

- [ ] 숫자는 한국 표기법으로 포맷된다 (억/백만원, 천 단위 콤마)
- [ ] 데이터가 없는 섹션은 렌더링하지 않는다 (빈 공간 없음)
- [ ] 전년 대비 상승은 파랑, 하락은 빨강으로 표시된다

### 인터랙션

- [ ] 멘트 복사 버튼 클릭 시 "복사됨" 피드백이 표시된다
- [ ] 제안 포인트 카드 클릭 시 모달이 열린다
- [ ] 기업 히스토리 "자세히 보기" 클릭 시 모달이 열린다
- [ ] 영업 멘트 WWH 토글이 접기/펼치기로 동작한다
- [ ] 즐겨찾기 별 클릭 시 상태가 시각적으로 즉시 변경된다
- [ ] 사이드바 검색이 실시간으로 필터링된다

### 반응성

- [ ] 데스크톱(1280px+)에서 레이아웃이 깨지지 않는다
- [ ] 모든 버튼과 링크가 클릭 가능한 영역이 충분하다 (최소 28px)

## 보안 품질

배포 전 반드시 확인. 항목 하나라도 실패하면 배포 금지.

- [ ] 모든 API Route에서 `auth.getUser()`로 세션을 검증한다
- [ ] `analyses` 테이블에 RLS가 활성화되어 있다
- [ ] Supabase Storage 버킷에 사용자별 접근 정책이 설정되어 있다
- [ ] `SUPABASE_SERVICE_ROLE_KEY`가 클라이언트에 노출되지 않는다
- [ ] `ANTHROPIC_API_KEY`가 클라이언트에 노출되지 않는다
- [ ] `.env.local`이 `.gitignore`에 포함되어 있다
- [ ] 데이터 조회 시 `user_id` 조건이 포함되어 있다

---

## 코드 품질

- [ ] `any` 타입이 사용되지 않는다
- [ ] 인라인 `style` 속성으로 색상이 하드코딩되지 않는다
- [ ] `console.log`가 커밋에 포함되지 않는다
- [ ] 빈 `catch` 블록이 없다 (최소한 `console.error` 포함)
- [ ] 컴포넌트 파일명이 PascalCase를 따른다
- [ ] 환경변수가 `.env.local`에서만 관리된다

---

## AI 분석 품질

분석 결과를 직접 확인하는 항목.

- [ ] 회사 요약이 실제 PDF 데이터와 일치한다
- [ ] 이슈가 최소 1개 이상 도출된다
- [ ] 컨설팅 서비스 제안이 재무 데이터와 연결된다 (관련 없는 제안 없음)
- [ ] 영업 멘트 7단계가 모두 생성된다
- [ ] 4인 다각도 분석 각각의 의견이 2문장 이상이다
- [ ] JSON 파싱 오류 없이 `AnalysisResult` 타입이 완성된다
